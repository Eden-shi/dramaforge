# DramaForge · 短剧生成平台

端到端短剧生成平台：**主题 → 剧本 → 分镜 → 图/视频/配音素材 → 成片合成**。
对接国内主流 AI 平台（通义/智谱/DeepSeek/MiniMax/可灵/即梦 等）。

## 技术栈

- **全栈 TypeScript**（Node 后端 + React 前端），前后端类型共享
- 国内平台经 HTTP/REST 对接，多数走 OpenAI 兼容协议
- 架构按可演进成 SaaS 设计（Provider 抽象、存储抽象、任务队列）

## 目录结构

- `packages/shared` 前后端共享领域模型与类型
- `apps/server` Fastify 后端：API、Provider、管线、任务队列、存储
- `apps/web` React + Vite 前端

## 开发

- `npm install`

### 生产模式（单端口，推荐自用）

- `start.cmd` —— 自动构建并以 http://127.0.0.1:7800 同时提供 UI 与 API

### 开发模式（热重载，双端口）

- `dev.cmd` —— 后端 :7800（热重载）、前端 :5173（HMR）

## 架构要点（面向 SaaS 演进）

- **Provider 抽象层**：四类能力（LLM/图/视频/TTS）各一接口，新平台只需加一个 descriptor，不改管线与 UI。
- **管线编排**：主题→剧本(流式)→分镜→素材→成片，每步可单独触发、结果可审阅编辑。
- **任务队列**：进程内串行实现（接口稳定），演进时换 Redis/BullMQ 即可。
- **存储抽象**：当前 JSON 文件（零原生依赖），接口稳定，可换 SQLite/Postgres。
- **单端口部署**：后端直接托管编译后的前端，自用最省事；多用户时前后端可拆分。
